08 / 12
Database

Database Schema Design

Table Key Columns Relations Purpose
👤 users id (UUID), name, email, business_name, business_type, phone, city Primary table User authentication & profile
📦 products id, name, category, price, stock, sku, status user_id → users Inventory management
📋 orders id, customer_name, items (JSONB), total, status, payment user_id → users Order tracking
👥 customers id, name, email, phone, city, total_orders, total_spent user_id → users Customer management
📁 documents id, name, type, size, category, file_url user_id → users Document storage

Multi-tenant architecture: Har user ka data separate — user_id foreign key se linked. Supabase (PostgreSQL) pe hosted with Row Level Security.
